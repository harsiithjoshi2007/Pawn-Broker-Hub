Product Requirements Document (PRD)

Pawn Broker — Complete Gold & Silver Loan Management Web Application

Overview

App/Website Name: Pawn Broker



Pawn Broker is a comprehensive, full-stack web application designed exclusively for gold and silver loan businesses. It consolidates every operational aspect of a pawnbroking business — customer onboarding, jewellery valuation, loan lifecycle management, interest calculation, payment tracking, receipt generation, compliance documentation, and advanced reporting — into a single, unified platform. The application provides role-based access for Admin, Manager, and Staff users, ensuring that each team member interacts only with the functionality appropriate to their role. With automated reminders, professional PDF receipts, and real-time dashboard analytics, Pawn Broker eliminates manual record-keeping and reduces the risk of errors in day-to-day loan operations.



Target Audience: Independent pawn shop owners, gold loan business operators, jewellery loan managers, and their operational staff across single and multi-branch setups. The application is designed for users with varying levels of technical proficiency, prioritizing a clean, intuitive interface that non-technical staff can confidently operate. The core problems it solves include fragmented record-keeping across paper and spreadsheets, manual interest calculation errors, missing payment trails, lack of professional customer documentation, and the absence of centralized reporting for business decision-making.

Tech Stack

Front-End

Framework: React (with Vite as the build tool)

Styling: Tailwind CSS

UI Components: ShadCN UI component library

Charts: Recharts for all dashboard data visualizations

PDF Generation: React-PDF / jsPDF with html2canvas for receipt and report downloads

Excel Export: SheetJS (xlsx library)

Barcode/QR Code: react-qr-code and react-barcode libraries

Webcam Capture: react-webcam for customer photo and signature capture

Form Management: React Hook Form with Zod validation

State Management: Zustand for global client-side state

Date Handling: date-fns

Back-End

Platform: Supabase

Authentication: Supabase Auth with JWT tokens and role-based Row Level Security (RLS) policies

Database: Supabase PostgreSQL — structured relational tables for customers, loans, payments, renewals, jewellery items, notifications, audit logs, users, roles, settings, and documents

Storage: Supabase Storage buckets for customer photos, KYC documents, jewellery images, and company logos

Real-Time: Supabase Realtime subscriptions for live dashboard updates and notification delivery

Edge Functions: Supabase Edge Functions for server-side PDF generation triggers, scheduled reminders, and backup operations

APIs & Integrations

SMS Notifications: Twilio SMS API or MSG91 (configurable in settings)

WhatsApp Notifications: Twilio WhatsApp API or WATI (configurable in settings)

Email Notifications: Resend API or SendGrid for transactional emails

Market Gold/Silver Prices (optional): Metal price API (e.g., metals-api.com) for live market value suggestions during loan entry

Technical Requirements

Supabase RLS policies enforcing role-based data access at the database level

All file uploads stored in private Supabase Storage buckets with signed URLs

Service worker registered for PWA offline caching

All forms include client-side and server-side validation

Audit logging for all create, update, and delete operations stored in a dedicated audit_logs table

Environment variables managed securely via Vite .env configuration

User Flow

Flow 1: First-Time Admin Setup

Admin navigates to the web application URL and is redirected to /login

Admin logs in with pre-configured credentials

System redirects to /dashboard — dashboard shows empty states for all metrics

Admin is prompted with a setup banner: "Complete your company profile to get started"

Admin clicks the banner → navigates to /settings/company

Admin uploads logo, fills in company name, address, phone, GST number, and saves

Admin navigates to /settings/users and creates Manager and Staff user accounts, assigning roles and setting initial passwords

Admin navigates to /settings/loan-defaults and configures default interest rates and penalty rate

Admin navigates to /settings/notifications and enters SMS/WhatsApp/Email API credentials and sets reminder days

Setup complete — Admin returns to /dashboard

Flow 2: Adding a New Customer and Creating Their First Loan

Staff member logs in → lands on /dashboard

Staff clicks "New Loan" in sidebar or dashboard quick action

System navigates to /loans/new → Step 1 (Customer Selection) displayed

Staff types customer name in the search autocomplete — no result found

Staff clicks "Create New Customer" shortcut link

System opens /customers/new in a new tab (or inline modal)

Staff fills in customer name, phone, WhatsApp number, Aadhaar number, PAN number, and address fields

Staff uploads customer photo using webcam capture button — webcam modal opens, photo is captured and confirmed

Staff uploads Aadhaar front and back images via file upload zones

Staff clicks "Save Customer" → customer is saved, Customer ID auto-generated

System returns to loan creation form (Step 1) with the new customer auto-selected

Staff proceeds to Step 2 — Jewellery Details

Staff fills in jewellery type (e.g., "Necklace"), category ("Ornament"), gross weight (25g), stone weight (2g) — net weight auto-calculated as 23g

Staff selects purity: 22K Gold

Staff enters estimated value (₹60,000) and market value (₹62,000)

Staff uploads a jewellery photo

Staff clicks "Add Another Item" to add a second jewellery row and fills in details for a bangle

Staff clicks "Next" → proceeds to Step 3 — Loan Terms

Staff enters principal amount (₹50,000)

Staff sets interest rate: 2% per month, Simple Interest

Staff sets loan duration: 6 months

System instantly calculates and displays: Due Date (6 months from today), Total Interest (₹6,000), Total Payable (₹56,000)

Staff sets penalty rate: 1% per month

Staff clicks "Submit Loan"

System saves loan, auto-generates Loan Number (e.g., GL-2024-00001)
